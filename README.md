# Тестовое задание для "ITK academy"
***
Данное тестовое задание было выполнено в качестве одного из этапов собеседования на вакансию "**Junior Python Developer**".
Основные требования указанные в PDF файле:
1. Реализовать эндпоинт `POST api/v1/wallets/<WALLET_UUID>/operation`
Содержание тела запроса: 
```
{ 
  "operation_type": “DEPOSIT” or “WITHDRAW”, 
  "amount": 1000 
} 
```
Запрос в зависимости от "operation_type" выполняет соответствующую операцию изменяя значение в БД.
 
2. Реализовать эндпоинт для получения баланса кошелька `GET api/v1/wallets/{WALLET_UUID}`.
3. Соответствие **PEP8**.
4. Наличие миграций для БД.
5. Упаковка в контейнер и запуск через docker-compose/
6. Покрытие эндпоинтов тестами.
7. Доступ через github.

## Стек технологий
***
- Python 3.x
- Django
- Django REST Framework (DRF)
- PostgreSQL
- Docker
- Docker Compose

## Структура проекта
***
- `config/` - Настройки Django.
- `terminal/` - Приложение Django (модели, вьюхи, сериализаторы).
- `requirements.txt` - Зависимости Python.
- `Dockerfile` - Описание контейнера приложения.
- `docker-compose.yaml` - Оркестрация контейнеров (приложение, БД).
- `env.example` - Шаблон файла переменных окружения.

## Переменные окружения
***
В проекте используются переменные окружения. Для корректной работы приложения **обязательно** требуется создать файл
с переменными окружения `.env`. Шаблон для создания файла находится в файле `env.example`.

| Переменная окружения | Описание                                                                                            |
|----------------------|-----------------------------------------------------------------------------------------------------|
| `DEBUG`              | Включает/выключает режим отладки Django. При `True` показываются подробные ошибки.                  |
| `SECRET_KEY`         | Секретный ключ Django, используемый для подписи сессий, CSRF-токенов и других функций безопасности. |
| `POSTGRES_DB`        | Имя базы данных PostgreSQL, к которой будет подключаться приложение.                                |
| `POSTGRES_USER`      | Имя пользователя для подключения к базе данных PostgreSQL.                                          |
| `POSTGRES_PASSWORD`  | Пароль для пользователя `POSTGRES_USER`.                                                            |
| `DB_HOST`            | Адрес (имя хоста или IP) сервера базы данных PostgreSQL.                                            |
| `DB_PORT`            | Порт, на котором прослушивает PostgreSQL (обычно 5432).                                             |

## Поднятие проекта в Docker Compose
***
1. Убедитесь, что у вас установлены:
   - Docker.
   - Docker Compose (для Docker Desktop он включён по умолчанию).
2. Создайте файл `.env`:
   - Укажите все необходимые переменные окружения (например, SECRET_KEY, POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD, DB_HOST, DB_PORT).
3. Откройте терминал и перейдите в директорию, где находится файл `docker-compose.yaml`
4. Выполните команду: `docker-compose up --build`.
   - `up`: создаёт и запускает контейнеры, определённые в `docker-compose.yaml`.
   - `--build`: пересобирает образы перед запуском контейнеров (полезно при изменениях в Dockerfile или зависимостях).

Проект будет доступен по адресу http://localhost:8000.
